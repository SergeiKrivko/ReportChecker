Тебе будут переданы главы текста. Для каждой главы указано ее название и изменения в содержании:
- `+` - строка добавлена
- `-` - строка удалена
- `*` - строка изменена
- ` ` - без изменений
Также для каждой строки (кроме удаленных) указан ее номер.
Номера и отметки об изменении не содержатся в исходном тексте.

Также будет передан список найденных ранее ошибок (возможно, пустой)
и список инструкций, которые тебе нужно соблюдать 
(например, там может быть указано, что считать или не считать ошибкой).
Тебе нужно найти новые ошибки, появившиеся в добавленных/измененных фрагментах.

Ответ должен быть представлен в формате json в следующем виде:

${OutputExample}

`chapter`: название главы, в которой обнаружена ошибка,
`title`: короткое описание ошибки (форматирование html),
`priority`: От 1 до 10 (1 - наиболее критичные ошибки, 10 - незначительные предложения),
`comment`: расширенное описание ошибки (форматирование html).
`patch`: предложение по исправлению ошибки. Должно быть представлено в виде списка строк, где
    `number`: номер строки,
    `content`: новое содержание строки,
    `type`: `Add` - чтобы добавить строку после `number` (0, чтобы добавить в начало),
            `Delete` - чтобы удалить строку (в этом случае поле `content` не нужно),
            `Modify` - чтобы заменить содержимое строки.
            
`patch` должен быть указан для простых ошибок с тривиальным вариантом исправления.
Если не существует тривиального варианта исправления, поле может быть `null`.

ЕСЛИ изменения в тексте НЕ СОДЕРЖАТ новых ошибок, ВЕРНИ ПУСТОЙ СПИСОК (`[]`).
