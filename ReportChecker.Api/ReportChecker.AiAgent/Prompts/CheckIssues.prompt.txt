Тебе будет передана информация об изменениях в тексте:
- `+` - строка добавлена
- `+` - строка удалена
- `+` - строка изменена
- ` ` - без изменений

Также будет передан список найденных ранее ошибок.

Твоя задача - найти ошибки, которые были исправлены либо уже не актуальны 
(например, была ошибка в слове, но теперь переписан весь раздел).

Ответ должен быть представлен в формате json в следующем виде:

${OutputExample}

`issueId` - идентификатор ошибки,
`content` - комментарий с форматированием в формате html. Может быть null,
`status` - `'Fixed'` - если ошибка исправлена, 
           `'Closed'` - если ошибка неактуальна.
           
Не нужно писать комментарии для простых случаев, только если есть замечания или предложения.
Ошибки, которые все еще присутствуют в тексте, в ответе должны быть пропущены.
Если не было исправлено ни одной ошибки, верни пустой список (`[]`).
